# СИЗ.ПРО — Подбор фильтров СИЗОД

MVP сервиса подбора СИЗОД по составу воздуха рабочей зоны.
Подбор основан на базе веществ из СанПиН 1.2.3685-21 (2473 вещества) с
классификацией по маркам фильтров согласно ГОСТ 12.4.235-2019, дополненной
синонимами и экспертной разметкой "полумаска/маска" из 332 веществ (источник:
внутренняя база 3M/iNEX).

## Структура проекта

```
siz-pro/
├── index.html              — главная страница (вёрстка)
├── css/
│   └── style.css            — все стили
├── data/
│   └── substances.json      — база веществ (2473 шт.)
├── js/
│   ├── recommendation.js     — ЛОГИКА ПОДБОРА (марка фильтра, лицевая часть, FFP)
│   ├── search.js             — поиск/автодополнение по базе
│   ├── render.js             — отрисовка списка веществ и результата
│   └── app.js                — точка входа, обработка событий UI
└── README.md                 — этот файл
```

## Принцип разделения (важно для расширения)

- **`recommendation.js`** — чистая логика без DOM. Принимает список веществ
  с концентрациями, возвращает объект результата. Легко тестировать и
  переиспользовать (например, для будущего подбора по профессии из 767н).
- **`search.js`** — работа с базой данных, ничего не знает про UI.
- **`render.js`** — только отрисовка, ничего не знает про логику подбора.
- **`app.js`** — связывает всё вместе, обрабатывает события.

Такое разделение позволяет менять одну часть без риска поломать другие.
Например, чтобы изменить алгоритм подбора FFP — трогаем только
`recommendation.js`.

## Структура записи в substances.json

```json
{
  "id": 4,
  "name": "Азота диоксид (азот (IV) оксид; азота двуокись)",
  "cas": "10102-44-0",
  "formula": "NO2",
  "pdk": "2",            // исходная строка ПДК (может быть "X/Y" — разовая/среднесменная)
  "pdk_value": 2.0,       // распарсенное значение для расчётов (используется среднесменная)
  "state": "п",           // п = пары, а = аэрозоль, "п + а" = пары и аэрозоль
  "hazard_class": "3",
  "notes": "О",           // О = раздражающее действие (используется как fallback)
  "filter_mark": "NOP3",  // марка(и) фильтра, через запятую если несколько
  "synonyms": ["Оксид азота", "пероксид азота"],
  "face_piece": "half"    // "half" | "full" — определяет лицевую часть
}
```

## Логика подбора (recommendation.js)

1. Для каждого вещества определяется марка фильтра (`filter_mark` из базы).
2. Марки объединяются в итоговое обозначение фильтра.
   - Если есть и газы/пары, и аэрозоли — газовые марки + "P" без указания
     класса предфильтра (P1/P2/P3 выбирается пользователем самостоятельно).
   - Если только газы/пары — марки без P.
   - Если только аэрозоль — просто "P".
3. Лицевая часть: если хотя бы одно вещество имеет `face_piece: "full"` —
   требуется полнолицевая маска, иначе полумаска.
4. Кратность превышения ПДК = концентрация / `pdk_value` (среднесменная).
5. FFP-рекомендация (только для чистого аэрозоля, марка только "P"):
   - ≤4 ПДК → FFP1
   - ≤12 ПДК → FFP2
   - ≤50 ПДК → FFP3
   - >50 ПДК → FFP недостаточно, нужен сменный фильтр P3
6. Доп. опция "угольный слой": если есть аэрозоль + слабые газы/пары
   (кратность газов ≤1) — предлагается как альтернатива.

## Известные ограничения MVP (точки для будущих доработок)

- **332 вещества** имеют точную разметку `face_piece` из 3M; для остальных
  **2141** используется грубое правило: метка "О" (раздражающее действие)
  в СанПиН → требуется маска. Это консервативная оценка, может потребовать
  уточнения для конкретных веществ.
- **55 веществ** не имеют числового значения ПДК (`pdk_value: null`) —
  в основном редкие фармпрепараты/цитостатики. Для них кратность превышения
  не рассчитывается.
- Для комбинации "аэрозоль + газы/пары" класс предфильтра P1/P2/P3 не
  определяется автоматически — оставлено на решение пользователя.
- Источники синонимов ограничены пересечением с базой 3M (332 из 2473).
  Для часто запрашиваемых веществ можно добавлять синонимы вручную в
  `substances.json` (поле `synonyms`).

## Как добавить новое вещество или исправить данные

Откройте `data/substances.json`, найдите/добавьте запись по образцу выше.
Поле `pdk_value` нужно посчитать вручную (см. правило выше — среднесменная).

## Как изменить логику подбора

Все правила — в `js/recommendation.js`. Константы в начале файла
(`FFP_THRESHOLDS`, `FILTER_ORDER`, `FILTER_INFO`) удобно менять без
переписывания логики.

## Локальный запуск

```bash
cd siz-pro
python3 -m http.server 8000
# открыть http://localhost:8000
```

## Публикация на GitHub Pages

1. Создать репозиторий, загрузить содержимое папки `siz-pro/` в корень.
2. В настройках репозитория → Pages → Source → выбрать ветку `main`,
   папку `/ (root)`.
3. Сайт станет доступен по адресу `https://<username>.github.io/<repo>/`.
4. Для подключения домена `siz.pro` — добавить файл `CNAME` с содержимым
   `siz.pro` в корень репозитория и настроить DNS-записи у регистратора
   (A-записи на IP GitHub Pages, см. документацию GitHub).
